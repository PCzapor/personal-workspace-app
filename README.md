Personal Workspace App
Mały projekt full-stack zrobiony głównie do nauki i ogarnięcia tego, jak naprawdę działa aplikacja webowa od środka.
UI jest tu tylko środkiem — największy nacisk był na backend, auth i komunikację między warstwami.

Po co powstał ten projekt?
Chciałem przejść przez proces budowy aplikacji, w której:
frontend nie jest tylko widokiem,
backend robi realną robotę,
a autoryzacja nie jest “magiczną biblioteką”, tylko czymś, co rozumiem.

Na czym był fokus
Auth i sesje (główny temat)
Najwięcej uwagi poszło w:
logowanie użytkownika
obsługę sesji
chronione trasy
flow requestów wymagających uwierzytelnienia
Chodziło o zrozumienie mechanizmu, nie tylko użycie gotowego rozwiązania.

Frontend ↔ Backend + proxy
Projekt był poligonem do ogarnięcia:
jak frontend gada z backendem
po co jest proxy
jak wygląda sensowna struktura API
co powinno dziać się na serwerze, a co w UI

Backend podstawy
Backend był tu ważny, bo:
trzyma logikę,
obsługuje dane użytkownika,
oddziela odpowiedzialności od frontendu.
